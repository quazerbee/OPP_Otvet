# Лекция 1. Entity Framework Core и UML-диаграммы

## 1.1 Определения: EF, EF Core, ORM

> **Entity Framework** — представляет ORM (object-relational mapping) технологию отображения данных на реальные объекты от компании Microsoft и технологию доступа к этим данным.

> **Entity Framework Core** — позволяет абстрагироваться от самой базы данных и её таблиц и работать с данными как с объектами классов независимо от типа хранилища.

> **ORM (Object-Relational Mapping)** — технология отображения объектов на реляционные таблицы БД.

**Два уровня работы:**

| Уровень | Чем оперируем |
|---|---|
| Физический | таблицы, индексы, первичные и внешние ключи |
| Концептуальный (EF Core) | работаем с объектами |

**Ключевое преимущество:** одинаковый API для всех БД — менять СУБД можно без изменения бизнес-логики.

## 1.2 Платформы и СУБД

**Технологии стека .NET:** Windows Forms (WinForms), консольные приложения, WPF, UWP, ASP.NET Core.

**Операционные системы:** Windows, Linux, Mac OS X.

**Поддерживаемые СУБД:**

| Источник | СУБД |
|---|---|
| От Microsoft (встроенные провайдеры) | MS SQL Server, SQLite, PostgreSQL |
| Сторонние поставщики | MySQL |
| Облачные | Azure Cosmos DB (`Microsoft.EntityFrameworkCore.Cosmos`) |

**Определения платформ (для теста):**

- **Windows Forms (WinForms)** — платформа пользовательского интерфейса для создания классических приложений Windows с помощью визуального конструктора в Visual Studio.
- **WPF (Windows Presentation Foundation)** — аналог WinForms, система для построения клиентских приложений Windows с визуально привлекательными возможностями; графическая (презентационная) подсистема в составе .NET Framework.
- **UWP (Universal Windows Platform)** — унифицированная платформа для создания и запуска приложений в Windows 10 и Windows 10 Mobile.

## 1.4 Сущность, связи, DbContext

> **Сущность (Entity)** определяет набор данных, которые связаны с определённым объектом.

- Работа не с таблицами, а с объектами и их коллекциями.
- Свойства бывают **простые** (`int`, `string`) или **комплексные** (другие типы).
- Имеют одно или несколько уникальных свойств = **ключи**.

**Три типа связей (ассоциаций):**

| Связь | Описание |
|---|---|
| One-to-Many (1:M) | Один `College` имеет много `Students` |
| One-to-One (1:1) | Один `Student` связан с одним `Profile` |
| Many-to-Many (M:M) | Один `Student` посещает много `Courses` |

Механизм связи — **внешние ключи**.

```csharp
public class CollegeDbContext : DbContext
{
    public DbSet<College> Colleges { get; set; }
    public DbSet<Student> Students { get; set; }
}
```

- **DbSet** — набор сущностей определённого типа (как таблица в коде).
- **DbContext** — класс контекста данных для работы с сущностями.
- Принцип: 1 таблица (напр. `users`) → 1 класс в .NET (напр. `User`).

## 1.5 Архитектура EDM

Слои снизу вверх:

1. **База данных** — SQL DB / Other DB
2. **ADO.NET Data Provider** — провайдеры для конкретных СУБД
3. **Entity Framework Core** — Entity Client Data Provider, формирование SQL, Object Services, LINQ to Entities, Entity SQL, отслеживание состояния объектов
4. **Entity Data Model (EDM)** — три компонента:

| Компонент EDM | Что описывает |
|---|---|
| **Conceptual Model** | как мы видим данные (объекты) |
| **Mapping Model** | как объекты отображаются на таблицы |
| **Storage Model** | как данные хранятся в БД |

5. **Application UI Layer** — WinForms / WPF / Web App

## 1.6 Три подхода: Code / Database / Model First

| Подход | Начало | Процесс | Когда использовать |
|---|---|---|---|
| **Code First** | Пишешь классы C# | Классы → БД | Нет готовой БД, разработчики C# |
| **Database First** | Готовая БД | БД → EDMX → Классы | Есть готовая БД, знание SQL |
| **Model First** | EDMX-диаграмма | Диаграмма → Классы + БД | Системные интеграторы |

**Code First:** сначала пишется код на C#, потом по нему создаётся БД. Маппинг представлен в коде (convention + mapping API). Часто называется «code only». Если БД нет — EF может создать/удалить/пересоздать её при изменениях модели.

**Database First:** для тех, кто хорошо знает SQL. Создаётся готовая БД → генерируется EDMX-модель. Файл `.edmx` — XML с информацией о структуре БД, моделью данных и маппингом. Инструмент — графический дизайнер EF в Visual Studio.

**Model First:** создаётся графическая EDMX-модель в дизайнере → генерируются классы C# и БД. Генерируется **DDL** (data definition language) для создания БД. Преимущество — абстракция и от SQL, и от C#.

## 1.7 Миграции и LINQ

**Миграции.** Проблема: при изменении класса модели приходится удалять БД → потеря данных. Решение:

> Функция миграции **позволяет последовательно применять изменения схемы к базе данных, чтобы синхронизировать её с моделью данных** (без потери данных).

**LINQ.**

> **LINQ (Language Integrated Query)** — это технология, которая представляет собой набор функций в .NET, позволяющих писать структурированные запросы к базе данных.

> **LINQ to Entities** — LINQ использует похожие на SQL выражения языка C# для получения данных. EF Core транслирует выражение LINQ в SQL-запрос.

```csharp
var students = dbContext.Students
    .Where(s => s.College.Name == "MIT")
    .ToList();
// → SELECT * FROM Students WHERE CollegeId IN
//   (SELECT CollegeId FROM Colleges WHERE Name = 'MIT')
```

## 1.8 UML: типы отношений, видимость, множественность

**Структура класса на диаграмме** — прямоугольник из 3 отсеков:

| Отсек | Содержимое |
|---|---|
| 1 (верхний) | Имя класса |
| 2 (средний) | Атрибуты |
| 3 (нижний) | Операции (поведение) |

**Пять типов отношений:**

| Отношение | Графика | Описание |
|---|---|---|
| **Наследование (Generalization)** | сплошная линия, полая стрелка △ | Отношение «есть-а» (is-a); абстрактный класс — имя курсивом |
| **Ассоциация (Association)** | сплошная линия | Структурная связь между двумя одноранговыми классами |
| **Агрегация (Aggregation)** | сплошная линия, незакрашенный ромб ◇ | «Часть-целое»; части существуют **независимо**, разное время жизни |
| **Композиция (Composition)** | сплошная линия, закрашенный ромб ◆ | Частный случай агрегации; части **не могут** существовать без целого, живут и умирают вместе |
| **Зависимость (Dependency)** | пунктир ┈┈, открытая стрелка ▶ | Изменения в одном классе влияют на другой (но не наоборот) |

**Роль (Role)** — направленная цель ассоциации; пишется на концах ассоциативной линии.

**Видимость атрибутов и операций:**

| Символ | Тип | Уровень доступа |
|---|---|---|
| `+` | public | Общедоступные |
| `–` | private | Только класс |
| `#` | protected | Класс и наследники |
| `~` | package | Только пакет |

**Множественность (Cardinality / Multiplicity):**

| Обозначение | Значение |
|---|---|
| `1` | Ровно один |
| `0..1` | Ноль или один |
| `*` или `0..*` | Ноль или много |
| `1..*` | Один или много |
| `3..4` | Точно от 3 до 4 |
| `6` | Ровно 6 |
| `0..1, 3..4, 6..*` | Сложное (любое кол-во, кроме 2 или 5) |

> Множественность отражает, сколько объектов каждого класса принимают участие в отношениях.

## 1.9 Перспективы диаграмм и примеры

**Три перспективы применения диаграмм классов:**

| Перспектива | Что моделируем | Зависимость от языка |
|---|---|---|
| **Концептуальная** | Вещи реального мира (домен) | Независима |
| **Спецификация** | Программные интерфейсы и абстракции (без реализации) | Без привязки |
| **Реализация** | Конкретная реализация в коде (C#, Java…) | Зависима |

**Рекомендации по диаграммам:** вместо одной большой диаграммы с 20 классами — несколько маленьких; каждая = отдельная часть системы → улучшает читаемость.

**Инструмент:** **Visual Paradigm Online (VP Online) Free Edition** — бесплатное онлайн-ПО для диаграмм классов, других UML-диаграмм, ERD и организационных диаграмм.

**Пример 1 — Система заказов:** `Customer` (1) → `Order` → `OrderDetail` → `Item`; абстрактный `Payment` (amount) наследуется `Cash`, `Check`, `Credit`. Методы `Order`: `calcSubTotal()`, `calcTax()`, `calcTotal()`.

**Пример 2 — Графический интерфейс:** `Window` агрегирует `Shape` (абстрактный → `Circle`, `Rectangle`, `Polygon`); `Circle` → `Point` композиция; `Window` ← `Event` зависимость.

## 1.10 Создание модели в ASP.NET (16 шагов)

1. ПКМ на папке **Models** → Add → **ADO.NET Entity Data Model**; имя `College.edmx`.
2. В мастере выбрать **Empty Model**.
3. Открыть конструктор → ПКМ → **Properties**.
4. **Entity Container Name** = `CollegeDbContext`. Прочее: Code Generation Strategy `T4`, Database Generation Strategy `TablePerTypeStrategy.xaml`, DDL Template `SSDLToSQL10.tt`, Lazy Loading `True`, Namespace `CollegeModel`, Pluralize New Objects `True`.
5. ПКМ → Add New → **Entity**.
6. Первая сущность: Entity name `College`, Entity Set `Colleges`, ✓ Create key property — `CollegeId`, тип `Int32`.
7. ПКМ на `College` → Add New → **Scalar Property**.
8. Добавить свойство `Name`; создать вторую сущность `Student` (`StudentId`, `Int32`).
9. ПКМ → Add New → **Association**.
10. Настройка ассоциации:

| Параметр | Значение |
|---|---|
| Association Name | `CollegeStudent` |
| End 1 Entity / Multiplicity | `College` / `1` (One) |
| End 2 Entity / Multiplicity | `Student` / `*` (Many) |
| Navigation Property | ✓ `Students` и ✓ `College` |
| Add foreign key properties | ✓ |

11. ПКМ → **Generate Database from Model**.
12. Выбрать **New Connection**.
13. Data source: `Microsoft SQL Server (SqlClient)`; Server: `(localdb)\v11.0`; **Windows Authentication**; Database name: `ModelFirst.College`; ✓ Save in Web.Config as `CollegeDbContext`.
14. Next → дизайнер вычисляет SQL-скрипт.
15. Finish → откроется `CollegeModel.edmx.sql` → ПКМ → **Execute** (Ctrl+Shift+E).
16. Build → конструктор создаёт классы `College` и `Student` в папке Models.

# Лекция 2. Введение в ASP.NET Core

**Традиционная архитектура Web (CGI):**

- **Front End** (видит пользователь): HTML, CSS, JavaScript.
- **Back End** (сервер): App Logic (PHP, JS, Python, Java), Web Server, File System, Database (MySQL, PostgreSQL, MariaDB).
- Взаимодействие: Front End ⇄ (Request / Response) ⇄ Back End.

## 2.1 Что такое ASP.NET Core

> **ASP.NET Core** — представляет технологию для создания веб-приложений на платформе .NET, развиваемую компанией Microsoft.

- Языки: **C#** и **F#**.
- **Opensource** (открытый исходный код).
- Исходники на GitHub: `https://github.com/dotnet/aspnetcore/`.

## 2.2 Архитектура: Unified Web Framework

| Уровень | Компоненты |
|---|---|
| **Web UI** | MVC, Razor Pages, SPA, Blazor |
| **Services** | HTTP APIs, SignalR, gRPC |
| **Extensions** | Logging, Configuration, Options, File Providers, Dependency Injection |
| **Middleware** | Routing, Security, Localization, Caching, Compression, Session, Health Checks |
| **Servers** | Kestrel, IIS, HTTP.sys |
| **Worker Services** | фоновые сервисы |
| **Hosting** | хостинг (самый нижний уровень) |

## 2.3 Web UI: MVC, Razor Pages, SPA, Blazor

**Четыре модели Web UI:** MVC, Razor Pages, SPA, Blazor.

### MVC (Model-View-Controller)

> **ASP.NET MVC** — многофункциональная платформа для создания веб-приложений и API-интерфейсов с помощью структуры проектирования Model-View-Controller.

Поток: `Входящий HTTP-запрос → Контроллер → (Модель) → Представление → Ответ клиенту`.

| Компонент | Функции |
|---|---|
| **Model** | Состояние приложения и бизнес-логика; ViewModel — типы для данных представления |
| **View** | Представление содержимого через UI; используют Razor; минимум логики |
| **Controller** | Управление взаимодействием, работа с моделью, выбор представления; отправная точка |

### Razor Pages

> **Razor Pages** — это новая улучшенная версия MVC.

- Файлы `.cshtml` (смесь C# и HTML).
- За обработку запроса отвечают сущности — страницы Razor Pages.

### SPA (Single Page Application)

- Для **Angular** и **React**, размещённых на сервере .NET. При публикации файлы копируются в папку **wwwroot**.
- **React** — библиотека JavaScript для создания внешних UI (только представления).
- **Angular** — фреймворк JavaScript (привязка данных, маршрутизация, анимация).

| Traditional Page Lifecycle | SPA Lifecycle |
|---|---|
| Initial GET → HTML | Initial GET → HTML |
| Form POST → HTML | AJAX → JSON |

### Blazor

> **Blazor** — фреймворк, который позволяет создавать интерактивные приложения как на стороне сервера, так и на стороне клиента, и позволяет задействовать на уровне браузера низкоуровневый код WebAssembly.

| Подсистема | Описание |
|---|---|
| **Blazor Server** | Серверные приложения, поддерживается ASP.NET Core; связь с клиентом через **SignalR** |
| **Blazor WebAssembly** | Одностраничные интерактивные приложения клиентской стороны; запускаются в браузере через WebAssembly; `blazor.webassembly.js` загружает среду выполнения .NET |

> **Razor** — это синтаксис разметки для внедрения кода на основе .NET в веб-страницы. Состоит из: разметки Razor + C# + HTML. Файлы — `.cshtml`, компоненты — `.razor`.

## 2.4 Web API, SignalR, gRPC

> **Web API (Web Service API)** — это интерфейс обработки приложений между веб-сервером и веб-браузером.

⚠️ **Все веб-сервисы являются API, но не все API являются веб-сервисами.**

> **SignalR Core** — библиотека от Microsoft для создания приложений, работающих в режиме реального времени.

- Используется с ASP.NET Core; **двунаправленная связь**.
- API для RPC (вызов JS-функций в браузерах из серверного .NET кода).
- Где используется: чаты, соцсети, игры, карты, аукционы, голосования, панели управления, мониторинг данных.

> **gRPC** — фреймворк, использующий протокол RPC (Remote Procedure Call) для обмена сообщениями между клиентом и сервером.

- Цель — высокая производительность в критичных условиях.
- gRPC **НЕ является частью** ASP.NET Core или .NET.
- Создан **Google**, открытый код, часть экосистемы **CNCF** (Cloud Native Computing Foundation).
- Интегрирован в SDK для **.NET Core 3.0+**.

## 2.5 Серверы: Kestrel, IIS, HTTP.sys

| Сервер | Особенности |
|---|---|
| **Kestrel** | Кроссплатформенный веб-сервер. Включается по умолчанию. Поддерживает HTTPS, HTTP/2 (кроме macOS), WebSocket, UNIX-сокеты за Nginx. Прокси: IIS, Apache, Nginx |
| **IIS** (Internet Information Services) | В ASP.NET Core роль сводится к прокси. Хостинг через нативный модуль `AspNetCoreModule` → перенаправляет запросы на Kestrel → управляет запуском `dotnet.exe` |
| **HTTP.sys** | Веб-сервер, запускается **только в Windows**. Альтернатива Kestrel. Применяется: (1) нужно подключить сервер к Интернету напрямую без IIS; (2) нужна функция, отсутствующая в Kestrel |

## 2.6 Middleware, Worker Services, Hosting

> **Middleware** (промежуточное/связующее ПО) — это фрагмент кода в конвейере приложения, используемый для обработки запросов и ответов.

Примеры: аутентификация, обработка ошибок, обслуживание статических файлов (JS, CSS, изображения).

**Worker Services** — приложения, работающие как фоновый процесс (обработка задач по очереди/интервалам). Пример — службы Windows.

| Reg.ru | до 1 Гб | Windows Server 2012/2012 R2 | MS SQL Server 2012/2014 Web + MySQL |

# Лекция 3. Основы ASP.NET Core MVC

## 3.1 Три компонента MVC

> **Контроллеры (Controllers)** — классы, которые: обрабатывают запросы браузера; получают данные модели; вызывают шаблоны представления, которые возвращают ответ.

> **Представления (Views)** — компоненты, которые формируют пользовательский интерфейс приложения. Как правило, в UI отображаются данные модели.

> **Модели (Models)** — классы, представляющие данные в приложении. Используют логику проверки (бизнес-правила); объекты модели извлекают и сохраняют состояние в базе данных.

## 3.2 Создание проекта и структура

**Шаги:** Visual Studio → Создать проект → **Веб-приложение ASP.NET Core (Модель — представление — контроллер)** → имя `MvcMovie` → .NET 7.0, снять флажок «Не использовать операторы верхнего уровня».

⚠️ Регистр символов имени должен соответствовать `namespace`.

**Шаблоны проектов:** ASP.NET Core Web App (MVC), ASP.NET Core Empty, Blazor WebAssembly App, ASP.NET Core gRPC Service, Razor Class Library.

**Структура проекта:**

| Элемент | Назначение |

| **Controllers** | Контроллеры; по умолчанию `HomeController` |
| **Models** | Модели; по умолчанию `ErrorViewModel` |
| **Views** | Представления |
| **appsettings.json** | Конфигурация приложения |

Внутри **Views**: папка `Home` (`Index.cshtml`, `Privacy.cshtml`), папка `Shared` (`_Layout.cshtml`, `_ValidationScriptsPartial.cshtml`, `Error.cshtml`), `_ViewImports.cshtml`, `_ViewStart.cshtml`.

## 3.3 Контроллеры и методы

```csharp
public class HelloWorldController : Controller
{
    // GET: /HelloWorld/
    public string Index() => "This is my default action...";
    // GET: /HelloWorld/Welcome/
    public string Welcome() => "This is the Welcome action method...";
}
```

- `Index()` — HTTP GET, вызывается через `/HelloWorld/`.
- `Welcome()` — HTTP GET, вызывается через `/HelloWorld/Welcome/`.
- Контроллер наследуется от класса `Controller`.

## 3.4 Маршрутизация по умолчанию

```csharp
app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");
```

Если сегменты URL не указаны → по умолчанию контроллер `Home`, метод `Index`.

## 3.5 Параметры и строка запроса

```csharp
public string Welcome(string name, int numTimes = 1)
{
    return HtmlEncoder.Default.Encode($"Hello {name}, NumTimes is: {numTimes}");
}
```

Что делает (часто спрашивают):
1. Необязательный параметр C# (`numTimes = 1`).
2. `HtmlEncoder.Default.Encode` — защита от злонамеренного ввода (например, JavaScript).
3. Интерполированные строки `$"..."`.

**Строка запроса:** `…/Welcome?name=Rick&numtimes=4` → `Hello Rick, NumTimes is: 4`.
- Сегмент `Parameters` не используется; `?` — разделитель строки запроса; `&` разделяет пары «поле-значение».

## 3.6 Представления и Razor

> Шаблоны представлений на основе **Razor**: имеют расширение `.cshtml`; реализуют удобный способ создания выходных данных HTML с помощью C#.

> **Razor** — синтаксис разметки для внедрения кода на основе .NET в веб-страницы. Состоит из разметки Razor, C# и HTML.

```csharp
// было
public string Index() { return "This is my default action..."; }
// стало
public IActionResult Index() { return View(); }
```

- `View()` использует шаблон представления для создания HTML-ответа.
- **`IActionResult`** — интерфейс, представляющий результат метода действия.

**Поиск представления по умолчанию:** если имя не указано, MVC использует представление с именем метода (`Index` → `Index.cshtml`). Путь: `/Views/HelloWorld/Index.cshtml`.

## 3.7 Layout, секции, частичные представления

**Layout / мастер-страницы** позволяют задать единый шаблон для представлений (единообразный вид сайта, общие меню, стили, скрипты).

```csharp
builder.Services.AddControllersWithViews();
```

```cshtml
<title>METANIT.COM | @ViewBag.Title</title>
...
<div>@RenderBody()</div>
```

- `@ViewBag.Title` — передаёт заголовок из каждого представления.
- `@RenderBody()` — вставляет основное содержимое представлений.

**Секции:**

```cshtml
<footer>@RenderSection("Footer")</footer>      @* в layout *@
@section Footer { ... }                          @* в представлении *@
```

| Вариант | Синтаксис |
|---|---|
| Необязательная секция | `@RenderSection("Footer", false)` |
| Проверка наличия | `@if (IsSectionDefined("Footer")) { ... }` |

**Частичные представления (Partial Views):**

- Встраиваются в обычные представления; удобны для рендеринга AJAX-результата, панелей меню, блоков входа.
- Рендерит объект `PartialViewResult`, возвращаемый методом `PartialView()`.

| Версия `PartialView()` | Описание |
|---|---|
| `PartialView()` | представление с именем вызывающего метода |
| `PartialView(string? viewName)` | имя представления |
| `PartialView(object? model)` | передаёт данные `model` |
| `PartialView(string? viewName, object? model)` | имя + данные |

```cshtml
@await Html.PartialAsync("Hello")
```

`Html.PartialAsync()` — асинхронный, возвращает `IHtmlContent`, обёрнут в `Task<TResult>`.

## 3.8 Определение маршрутов

Поток: `Client → Http Request → Web Server → ASP.NET Routing Engine → Controller → View → Model → Http response`.

```csharp
app.MapControllerRoute(name: "default", pattern: "{controller}/{action}");
```

Параметры определяются в фигурных скобках `{имя}`.

| Тип | Пример |
|---|---|
| Дополнительные параметры | `{controller}/{action}/{id}` |
| Статический сегмент | `api/{controller}/{action}/{id}` |
| Необязательный параметр | `{controller}/{action}/{id?}` |
| Значения по умолчанию | `{controller=Home}/{action=Index}/{id?}` |

**Таблица сопоставления:**

| Запрос | Параметры |

| `…/Book` | controller=Book, action=Index |
| `…/Book/Show` | controller=Book, action=Show |
| `…/Book/Show/2` | controller=Book, action=Show, id=2 |

# Лекция 4. MVC: Tag-хелперы, формы, модели, EF Core

## 4.1 Tag-хелперы и addTagHelper

> **Tag-хелперы** представляют собой функциональность, предназначенную для генерации HTML-разметки.

- Применяются в представлениях; выглядят как html-элементы или атрибуты.
- Обрабатываются движком **Razor на стороне сервера** → преобразуются в стандартные html-элементы.

```cshtml
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers
<a asp-controller="Home" asp-action="Contacts">Контакты</a>
```

- 1-й параметр — какие хелперы доступны; `*` = все хелперы.
- 2-й параметр — библиотека (`Microsoft.AspNetCore.Mvc.TagHelpers`).
- ⚠️ Подключение определяется в файле **`_ViewImports.cshtml`**.

## 4.2 HTML-формы и их атрибуты

> HTML-форма позволяет пользователю вводить информацию для обработки. Задаётся элементом `<form>`.

| Атрибут | Назначение |
|---|---|
| `method` | Метод запроса: `GET` или `POST` |
| `action` | URL для обработки данных (абсолютный или относительный) |
| `enctype` | Кодирование данных. По умолчанию `application/x-www-form-urlencoded`; для файлов — `multipart/form-data` |
| `accept-charset` | Кодировка для обработки данных |
| `autocomplete` | Автозаполнение полей |
| `name` | Имя формы |
| `novalidate` | Отменяет встроенную проверку |
| `target` | Имя окна/фрейма для результата |

## 4.3 GET vs POST

> Главное отличие методов POST и GET — в способе передачи информации.

- **GET** — параметры через адресную строку (в HTTP-заголовке запроса).
- **POST** — параметры через тело HTTP-запроса, не отражаются в адресной строке.

## 4.4 Типы полей input

> `<input>` — самый разносторонний элемент, позволяет создавать разные элементы формы.

Основные атрибуты: `name`, `value`, `type`, `readonly`, `disabled`, `placeholder`.

| `type` | Назначение |
|---|---|
| `text` | Текстовое поле (атрибуты `maxlen`, `size`, `value`) |
| `password` | Поле с паролем |
| `radio` | Переключатель |
| `checkbox` | Флажок |
| `hidden` | Скрытое поле |
| `submit` | Кнопка отправки |
| `reset` | Кнопка очистки |
| `file` | Поле для файла (форма должна иметь `enctype="multipart/form-data"`) |
| `image` | Кнопка с изображением |
| `color` | Выбор цвета из палитры |
| `date` | Выбор даты |
| `datetime-local` | Дата с локальным временем |
| `month` | Месяц и год |
| `email` | E-mail (по умолчанию один; `multiple="multiple"` — несколько через запятую) |
| `number` | Числовое поле со стрелками |
| `range` | Ползунок (атрибуты `min` и `max`) |

## 4.5 select, textarea, label, datalist

```html
<select name="students" multiple>
    <option value="Helen" selected>Helen</option>
</select>
```
- `multiple` — множественный выбор; `selected` — заранее выбранный элемент.

```html
<textarea cols="30" rows="5"></textarea>
```
- `cols` — столбцы (ширина); `rows` — строки (высота).

- **`<label>`** — задаёт связь между элементом формы и его меткой; `for` у label = `id` у input.
- **`<fieldset>`** — группирование элементов формы.
- **`<datalist>`** — список вариантов при наборе в текстовом поле; `list` у input = `id` у datalist.

## 4.6 Tag-хелперы форм

| Атрибут | Назначение |
|---|---|
| `asp-action` | Действие контроллера |
| `asp-controller` | Контроллер для запроса |
| `asp-antiforgery` | `true` → генерируется antiforgery token |
| `asp-for` | Генерирует `<input>` для свойства модели |
| `asp-items` | Источник элементов для `<select>` |

```cshtml
@model MvcApp.Models.Product
<form asp-action="Create" asp-controller="Home" asp-antiforgery="true">
    <label asp-for="Name"></label>
    <input asp-for="Name" />
    <select asp-for="CompanyId" asp-items="ViewBag.Companies"></select>
</form>
```
В контроллере: `ViewBag.Companies = new SelectList(companies, "Id", "Name");`

## 4.7 Свои tag-хелперы

```csharp
public class TimerTagHelper : TagHelper
{
    public override void Process(TagHelperContext context, TagHelperOutput output)
    {
        output.TagName = "div";
        output.Content.SetContent($"Текущее время: {DateTime.Now:HH:mm:ss}");
    }
}
```
- Наследуется от `TagHelper`; переопределяется `Process(...)`.
- `output.TagName` — тег для замены; `output.Content.SetContent(...)` — содержимое.

**Асинхронный** — переопределяется `ProcessAsync`:
```csharp
var target = await output.GetChildContentAsync();
output.Content.SetHtmlContent("<h3>Общая информация</h3>" + target.GetContent());
```
`GetChildContentAsync()` возвращает HTML вложенных tag-хелперов (`SummaryTagHelper` = контейнер).

## 4.8 Модели MVC

> Ключевая задача моделей — описание структуры и логики используемых данных.

- Все модели оформляются как **POCO-классы** (plain-old CLR objects) — обычные классы C#.

**Три группы моделей:**
1. Объекты хранятся в хранилищах (БД, xml).
2. Модели представления (**ViewModel**) — передача данных в представление / из него.
3. Вспомогательные модели для промежуточных вычислений.

ViewModel позволяет передать в представление сразу несколько наборов данных.

## 4.9 EF Core, DbContext, Program.cs

> **Entity Framework** — ORM-решение, позволяющее автоматически связать обычные классы C# с таблицами в БД.

- Для MS SQL Server нужен пакет `Microsoft.EntityFrameworkCore.SqlServer` (добавляется через NuGet).

```csharp
public class ApplicationContext : DbContext
{
    public DbSet<User> Users { get; set; } = null!;
    public ApplicationContext(DbContextOptions<ApplicationContext> options) : base(options)
    {
        Database.EnsureCreated();   // создаёт БД при первом обращении
    }
}
```

`appsettings.json`:
```json
{ "ConnectionStrings": { "DefaultConnection": "Server=(localdb)\\mssqllocaldb;Database=usersdb;Trusted_Connection=True;" } }
```

`Program.cs`:
```csharp
string connection = builder.Configuration.GetConnectionString("DefaultConnection");
builder.Services.AddDbContext<ApplicationContext>(options => options.UseSqlServer(connection));
builder.Services.AddControllersWithViews();
app.MapDefaultControllerRoute();
```

Контекст в контроллер передаётся через конструктор (**Dependency Injection**).

## 4.10 CRUD-операции

```csharp
public async Task<IActionResult> Index() => View(await db.Users.ToListAsync());

[HttpPost]
public async Task<IActionResult> Create(User user)
{
    db.Users.Add(user);
    await db.SaveChangesAsync();
    return RedirectToAction("Index");
}
```

| Метод | Что делает |
|---|---|
| `ToListAsync()` | получает все объекты, передаёт список в представление |
| `Add(user)` | формирует SQL `INSERT` |
| `SaveChangesAsync()` | выполняет выражение в БД |
| `Remove(user)` | SQL `DELETE` |
| `Update(user)` | SQL `UPDATE` |
| `FirstOrDefaultAsync(...)` | поиск объекта по условию |
| `RedirectToAction("Index")` | перенаправление на метод |
| `NotFound()` | возврат, если объект не найден |

В представлении: `asp-route-id="@item.Id"` — передаёт `id` в маршрут.

## 4.11 Сортировка и фильтрация

```csharp
public enum SortState { NameAsc, NameDesc, AgeAsc, AgeDesc, CompanyAsc, CompanyDesc }
```

```csharp
IQueryable<User>? users = db.Users.Include(x => x.Company);
users = sortOrder switch {
    SortState.NameDesc => users.OrderByDescending(s => s.Name),
    SortState.AgeAsc   => users.OrderBy(s => s.Age),
    _                  => users.OrderBy(s => s.Name),
};
return View(await users.AsNoTracking().ToListAsync());
```

- `OrderBy` / `OrderByDescending` — сортировка; `Include(...)` — подгрузка связанных данных; `AsNoTracking()` — без отслеживания (производительность чтения).
- В представлении: `asp-route-sortOrder="@ViewBag.NameSort"`.

**Фильтрация:**
```csharp
if (company != null && company != 0) users = users.Where(p => p.CompanyId == company);
if (!string.IsNullOrEmpty(name))     users = users.Where(p => p.Name!.Contains(name));
companies.Insert(0, new Company { Name = "Все", Id = 0 });
```
- `Where(...)`, `Contains(...)`, `IsNullOrEmpty(...)`; `SelectList`; `Insert(0, "Все")` — элемент «Все» на первое место.

# Лекция 5. .NET MAUI — основы

## 5.1 Что такое .NET MAUI

> **.NET Multi-Platform App UI (.NET MAUI)** — кроссплатформенная платформа для создания собственных мобильных и классических приложений с помощью C# и XAML.

- Работает на **Android, iOS, macOS, Windows** из одной общей базы кода.
- Языки: **C#** и **XAML**; opensource.
- ⚠️ **MAUI = эволюция Xamarin.Forms** (расширена с мобильных до классических сценариев).

## 5.2 Компиляция и архитектура

| Платформа | Как компилируется |
|---|---|
| **Android** | C# → промежуточный язык (**IL**) → **JIT** в собственную сборку при запуске |
| **iOS** | Полностью заранее (**AOT**) → собственный код сборки **ARM** |

**Стек слоёв:** `App Code → .NET MAUI → (.NET for Android/iOS/Mac, WinUI 3) → .NET BCL → (Mono Runtime / .NET CoreCLR) → ОС`.

- ⚠️ **Mono Runtime** — для Android/iOS/macOS; **.NET CoreCLR** — для Windows.

**Кроссплатформенные API устройств:** датчики (акселерометр, компас, гироскоп), состояние сети, сведения об устройстве, буфер обмена, выбор файлов, безопасное хранение «ключ-значение», текст-в-речь.

## 5.3 Один проект, Hot Reload, инструменты

**Один проект (Single Project):** общий проект для всех платформ, один манифест (заголовок, идентификатор, версия), одна кроссплатформенная точка входа, общие файлы ресурсов.

**Hot Reload (горячая перезагрузка):**

| Тип | Что делает |
|---|---|
| **Горячая перезагрузка .NET** | изменение управляемого кода во время работы, без перекомпиляции |
| **Горячая перезагрузка XAML** | изменения XAML в работающем приложении; состояние и навигация сохраняются |

**Инструменты:** разрабатывать можно на Windows и MacOS, **НО** под iOS/macOS обязательно нужен **MacOS**. Среда — **Visual Studio 2022+**, на Mac — **Visual Studio for Mac**, также можно VS Code.

## 5.4 Шаблоны и структура проекта

| Шаблон | Назначение |
|---|---|
| **.NET MAUI App** | Стандартный проект |
| **.NET MAUI Blazor App** | Использует фреймворк Blazor |
| **.NET MAUI Class Library** | Библиотека классов |

**Структура проекта:**

| Элемент | Назначение |
|---|---|
| **Platforms** | Подкаталоги для каждой платформы (код взаимодействия с платформой) |
| **Resources** | Файлы ресурсов: шрифты, иконки, изображения (подпапки: AppIcon, Fonts, Images, Raw, Splash, Styles) |
| **App.xaml** | Ресурсы, общие для всего приложения |
| **App.xaml.cs** | Файл C#, с которого начинается выполнение |
| **AppShell.xaml** | Общий визуальный интерфейс многостраничного приложения |
| **AppShell.xaml.cs** | Программная логика, связанная с AppShell.xaml |
| **MainPage.xaml** | Визуальный интерфейс единственной страницы MainPage (XAML) |
| **MainPage.xaml.cs** | Логика страницы MainPage на C# |
| **MauiProgram.cs** | Класс MauiProgram — стартовый класс приложения (по умолчанию `App`) и общие настройки |

## 5.5 Порядок запуска и ключевые классы

**Цепочка запуска:** `Platforms → MauiProgram → App.xaml(.cs) → AppShell.xaml(.cs) → MainPage.xaml(.cs)`.

```csharp
public static class MauiProgram
{
    public static MauiApp CreateMauiApp()
    {
        var builder = MauiApp.CreateBuilder();
        builder.UseMauiApp<App>()
               .ConfigureFonts(fonts => fonts.AddFont("OpenSans-Regular.ttf", "OpenSansRegular"));
        return builder.Build();
    }
}
```

```csharp
public partial class App : Application   // реализует IApplication
{
    public App() { InitializeComponent(); MainPage = new AppShell(); }
}
public partial class AppShell : Shell    // оболочка для простых страниц, навигация
{
    public AppShell() { InitializeComponent(); }
}
```

- **MauiProgram** — стартовый класс (по умолчанию `App`).
- **App** : `Application`, реализует `IApplication`, устанавливает `MainPage = new AppShell()`.
- **AppShell** : `Shell` — оболочка для страниц, упрощает навигацию; распределён на два файла.

**ShellContent** (в `AppShell.xaml`): `Title` — заголовок; `ContentTemplate="{DataTemplate local:MainPage}"` — шаблон-страница; `Route="MainPage"` — главная страница при запуске.

## 5.6 XAML и пространства имён

> **XAML** представляет язык разметки на основе xml для создания объектов декларативным образом.

**Преимущества:** отделяет интерфейс от логики (дизайнеры/программисты работают автономно); более ясный и поддерживаемый код; интерфейс как набор страниц (подобно HTML).

| Пространство имён | Что определяет |
|---|---|
| `http://schemas.microsoft.com/dotnet/2021/maui` | Базовое — большинство типов .NET MAUI |
| `http://schemas.microsoft.com/winfx/2009/xaml` | Типы XAML и CLR; префикс `x:` |
| `xmlns:local="clr-namespace:HelloApp"` | Функционал текущего проекта |

⚠️ Только одно пространство имён может быть **базовым** (maui), остальные — с префиксом. `x:Class` указывает на класс страницы.

## 5.7 Android и activity

- **activity** — отдельный экран; то, что видим на экране Android-устройства.
- По умолчанию создаётся `MainActivity.cs`, наследуется от `MauiAppCompatActivity`.
- В `Platforms/Android`: Resources, AndroidManifest.xml, MainActivity.cs, MainApplication.cs.
- Установка устройства: Visual Studio → поле устройств → **Android Local Devices**.

## 5.8 Классы макетов (Layouts)

> Классы макетов .NET MAUI позволяют упорядочивать и группировать элементы управления.

| Макет | Описание |
|---|---|
| **StackLayout** | Одномерный стек по гориз./верт. Свойство `Orientation`, по умолчанию **Vertical** |
| **HorizontalStackLayout** | Горизонтальный стек, производительнее StackLayout |
| **VerticalStackLayout** | Вертикальный стек, производительнее StackLayout |
| **Grid** | Строки и столбцы; свойства `RowDefinitions` и `ColumnDefinitions` |
| **FlexLayout** | Как StackLayout, элементы по типу «плитка» |
| **AbsoluteLayout** | Размещение по явным/относительным значениям (от левого верхнего угла) |
| **ScrollView** | Прокручивает содержимое |

## 5.9 Свойства элементов: размеры, отступы, выравнивание

**Размеры:** `WidthRequest` (ширина), `HeightRequest` (высота).

**Отступы:**

| Свойство | Что определяет |
|---|---|
| **Margin** | Внешний отступ от других элементов/контейнера |
| **Padding** | Внутренний отступ от содержимого до границ |

**Выравнивание элемента** — базовый класс `View`, свойства `HorizontalOptions` / `VerticalOptions`:

| Значение | По горизонтали | По вертикали |
|---|---|---|
| Start | по левому краю | по верху |
| Center | по центру | по центру |
| End | по правому краю | по низу |
| **Fill** | заполняет всё пространство контейнера | — |

**Выравнивание текста** — `HorizontalTextAlignment` / `VerticalTextAlignment` (перечисление `TextAlignment`: Start / Center / End — **без Fill**).

## 5.10 Текст, кнопки, поля ввода

**Цвет:** `BackgroundColor` (фон), `TextColor` (текст); тип `Microsoft.Maui.Graphics.Color`.

**Шрифт:**

| Свойство | Тип |
|---|---|
| `FontFamily` | string |
| `FontSize` | double |
| `FontAttributes` | перечисление (Bold / Italic) |
| `FontAutoScalingEnabled` | bool, по умолчанию `true` |

**Кнопка (Button):** свойства `BorderColor`, `BorderWidth`, `LineBreakMode`, `ImageSource`, шрифтовые. События:

| Событие | Когда срабатывает |
|---|---|
| **Clicked** | нажатие; срабатывает, когда пользователь убирает курсор с кнопки |
| **Pressed** | курсор/мышь находятся на кнопке |
| **Released** | пользователь убрал курсор с кнопки |

```csharp
private void OnButtonClicked(object sender, EventArgs e)
{
    Button button = (Button)sender;   // приведение типа отправителя
    button.Text = "Нажато!";
}
```

**Текстовые поля:**

| Класс | Описание |
|---|---|
| **Entry** | однострочное текстовое поле |
| **Editor** | многострочное текстовое поле |

`Entry`: `Placeholder`, `MaxLength`. Свойство **`Keyboard`** — формат клавиатуры:

| Раскладка | Назначение |
|---|---|
| Default | по умолчанию |
| Text | текст |
| Chat | текст и эмодзи |
| Url | url-адреса |
| Email | электронные адреса |

**KeyboardFlags:** None, CapitalizeSentence (первое слово предложения), Spellcheck, Suggestions, CapitalizeWord (каждое слово), CapitalizeCharacter (каждый символ), CapitalizeNone, All.

**События Entry:** `TextChanged` (при вводе) и `Completed` (при завершении); тип аргумента — **`TextChangedEventArgs`**.

**Дизайн:** инструмент — **Figma**; типовые экраны: Welcome, Log In, Home, Slot, Calendar, Success, Error.

# Лекция 6. Элементы управления .NET MAUI

## 6.1 BoxView, ScrollView, Image

> **BoxView** представляет прямоугольную область. Применяется для создания окрашенных областей либо в качестве декоративного примитивного оформления.

| Свойство | Тип / по умолчанию |
|---|---|
| `Color` | структура Color |
| `CornerRadius` | float (радиус границы) |
| `WidthRequest` | по умолчанию **40** единиц |
| `HeightRequest` | по умолчанию **40** единиц |

**ScrollView:**

| Свойство | Тип |
|---|---|
| `Content` | View |
| `ContentSize` | Size (только чтение) |
| `Orientation` | ScrollOrientation, по умолчанию **Vertical** |

Перечисление `ScrollOrientation`: **Vertical** (по умолчанию), Horizontal, Both, Neither.

`ScrollToAsync` — две версии: `(double x, double y, bool animated)` и `(Element element, ScrollToPosition position, bool animated)`. `ScrollToPosition`: MakeVisible, Start, Center, End.

**Image** — форматы **png, jpg, gif, svg**:

| Свойство | Тип / по умолчанию |
|---|---|
| `Aspect` | режим масштабирования |
| `IsAnimationPlaying` | bool, false |
| `IsLoading` | bool, false |
| `IsOpaque` | bool, false |
| `Source` | ImageSource |

## 6.2 ImageSource и загрузка изображений

| Метод | Откуда / возвращает |
|---|---|
| `FromFile()` | локальный файл → **FileImageSource** |
| `FromUri()` | адрес Uri (сеть) → **UriImageSource** |
| `FromResource()` | ресурс сборки → **ResourceImageSource** |
| `FromStream()` | поток → **StreamImageSource** |

- **Локальный файл:** папка `Resources\Images`, Build action = **MauiImage**.
- **Сеть (Uri):** кэширование по умолчанию **1 день**; `CacheValidity` (TimeSpan), `CachingEnabled` (bool, по умолчанию `true`).
- **Встроенный ресурс:** Build Action = **Embedded resource**.

**Масштабирование (`Aspect`):**

| Значение | Что делает |
|---|---|
| AspectFit | сохраняет пропорции, в границах |
| AspectFill | сохраняет пропорции, заполняет всё |
| Center | центрирует, сохраняет пропорции |
| **Fill** | **НЕ** сохраняет пропорции (растягивает) |

## 6.3 Border, DatePicker, TimePicker

> **Border** выполняет декоративные функции (граница/фон). ⚠️ Вмещает **ТОЛЬКО ОДИН** элемент!

| Свойство | Тип / по умолчанию |
|---|---|
| `StrokeShape` | IShape |
| `Stroke` | Brush |
| `StrokeThickness` | double, **1.0** |
| `StrokeDashArray` | DoubleCollection |
| `StrokeLineCap` | PenLineCap, Flat |
| `StrokeLineJoin` | PenLineJoin, Miter |

**DatePicker:**

| Свойство | По умолчанию |
|---|---|
| `MaximumDate` | **31 декабря 2100** |
| `MinimumDate` | **1 января 1900** |
| `Date` | DateTime.Today |
| `Format` | **"D"** (расширенный) |

Событие `DateSelected`, аргумент `DateChangedEventArgs` (`e.NewDate`).

**TimePicker:** `Time` (TimeSpan, по умолчанию 0), `Format` = **"t"** (сокращённое). Обработка через `PropertyChanged` (`if (e.PropertyName == "Time")`).

## 6.4 Stepper, Slider, Switch, CheckBox, RadioButton

**Stepper** (выбор числа кнопками +/−):

| Свойство | По умолчанию |
|---|---|
| `Increment` | 1 |
| `Minimum` | 0 |
| `Maximum` | **100** |
| `Value` | 0 |

Событие `ValueChanged`, аргумент `ValueChangedEventArgs` (`e.NewValue`).

**Slider** (горизонтальный ползунок):

| Свойство | Тип / по умолчанию |
|---|---|
| `Minimum` | double, 0 |
| `Maximum` | double, **1** (НЕ 100!) |
| `Value` | 0 |
| `ThumbColor` | цвет указателя |
| `TrackColor` | цвет ползунка |

**Switch:** `IsToggled` (вкл/выкл), `ThumbColor`, `OnColor`.

**CheckBox:** `IsChecked` (отмечен/нет), `Color`.

**RadioButton:** `Content` (текст/View, справа от круга), `GroupName` (имя группы — выбор одной из группы), `Value` (object), `BorderColor`.

## 6.5 Picker, TableView, WebView

**Picker** (выпадающий список): `ItemsSource` (IList), `SelectedItem` (object), `CharacterSpacing` (double), `FontAttributes`. Событие `SelectedIndexChanged`.

**TableView:** `Intent`, `Root` (TableRoot), `RowHeight` (int). Структура: `TableView → TableRoot → TableSection → TextCell` (с `Text` и `Detail`).

**WebView** (мини-браузер): `Cookies` (CookieContainer), `CanGoBack` (bool), `CanGoForward` (bool), `Source` (WebViewSource).

## 6.6 Всплывающие окна, ProgressBar, ActivityIndicator

**Всплывающие окна** — методы у объекта `Page` (есть у любой страницы): окно-уведомление (Alert, кнопка OK), окно подтверждения (Да/Нет), окно выбора действия (ActionSheet).

**ProgressBar:** `Progress` (double, диапазон **0..1**, по умолчанию 0), `ProgressColor`.

**ActivityIndicator** (круговой): `IsRunning` (true → активен), `Color`.

## 6.7 Триггеры

> **Триггеры** позволяют декларативно задать действия, которые выполняются при изменении свойств визуального элемента, изменении данных или генерации событий.

Определяются: через коллекцию `Triggers` на уровне элемента или как ресурс на уровне страницы/приложения.

| Тип | Класс | Использует |
|---|---|---|
| Триггеры **свойств** | `Trigger` | объект **Setter** |
| Триггеры **событий** | `EventTrigger` | действие **TriggerAction** |
| Триггеры **данных** | `DataTrigger` | привязку и объект **Binding** |

## 6.8 ListView и типы ячеек

**ListView** связывается через `ItemsSource` → принимает `IEnumerable<T>`. Для сложных объектов — свойство `ItemTemplate` (объект `DataTemplate`).

| Тип ячейки | Что делает |
|---|---|
| **ViewCell** | настраивается по своему усмотрению |
| **TextCell** | заголовок + детальное описание |
| **ImageCell** | + изображение рядом с заголовком и описанием |

`TextCell`: `Text`, `Detail`, `TextColor`, `DetailColor`. ⚠️ **ImageCell расширяет (наследует) TextCell**, добавляя `ImageSource`.

`Header` и `Footer` у ListView — строки заголовка и футера; `RowHeight` — высота строк.

## 6.9 MVVM

> **MVVM (Model-View-ViewModel)** основывается на разделении функциональной части приложения на три компонента:

| Компонент | Определение |
|---|---|
| **View** | представление или пользовательский интерфейс |
| **Model** | модель или данные, используемые в приложении |
| **ViewModel** | промежуточный слой между представлением и данными, обеспечивает их взаимодействие |

- ViewModel реализует интерфейс **`INotifyPropertyChanged`**; событие `public event PropertyChangedEventHandler? PropertyChanged;`.
- Через свойство `BindingContext` устанавливается объект ViewModel (`BindingContext = new PersonViewModel();`).
- Привязка в XAML: `Text="{Binding Name}"`.

## 6.10 Навигация и NavigationPage

Для простых сценариев — тип страницы `NavigationPage`.

| Метод | Что делает |
|---|---|
| `Navigation.PushAsync(new CommonPage())` | переход на обычную страницу (в стек) |
| `Navigation.PopAsync()` | назад (из стека) |
| `Navigation.PushModalAsync(new ModalPage())` | на модальную страницу |
| `Navigation.PopModalAsync()` | возврат с модальной |

- Главной страницей должен быть `NavigationPage`, а не `ContentPage`: `MainPage = new NavigationPage(new MainPage());`.
- **Стек навигации (LIFO):** `Push` — добавляет страницу на вершину (вперёд), `Pop` — убирает (назад).
- Передача данных — через конструктор страницы: `await Navigation.PushAsync(new PageB(56));`.
- Заголовок — свойство `Title` (отображается в шапке NavigationPage).

# ⚠️ Пары-«ловушки» — что чаще всего путают

| Пара | Различие |
|---|---|
| **Агрегация ◇ vs Композиция ◆** | Агрегация — части независимы; композиция — части умирают вместе с целым |
| **Code / Database / Model First** | начинаем с кода C# / с готовой БД / с EDMX-диаграммы |
| **React vs Angular** | React — библиотека; Angular — фреймворк |
| **Blazor Server vs WebAssembly** | сервер (через SignalR) vs клиент в браузере (WebAssembly) |
| **Все веб-сервисы — API, но не все API — веб-сервисы** | формулировка для теста |
| **GET vs POST** | GET — адресная строка; POST — тело запроса |
| **enctype** | по умолчанию `application/x-www-form-urlencoded`; для файлов `multipart/form-data` |
| **Margin vs Padding** | внешний vs внутренний отступ |
| **HorizontalOptions vs HorizontalTextAlignment** | выравнивание элемента (есть Fill) vs выравнивание текста (без Fill) |
| **Clicked / Pressed / Released** | убрал курсор / курсор на кнопке / убрал курсор |
| **Entry vs Editor** | однострочное vs многострочное поле |
| **Android vs iOS компиляция** | Android → IL + JIT (при запуске); iOS → AOT → ARM |
| **Mono Runtime vs .NET CoreCLR** | Android/iOS/macOS vs Windows |
| **Stepper vs Slider** | Stepper: Max=100, есть Increment; Slider: Max=1, без Increment |
| **Switch vs CheckBox** | `IsToggled` (+ThumbColor/OnColor) vs `IsChecked` (+Color) |
| **ProgressBar vs ActivityIndicator** | горизонтальная полоса (Progress 0..1) vs круговой (IsRunning) |
| **MVC vs MVVM** | Controller vs ViewModel (промежуточный слой) |
| **Триггеры** | Trigger→Setter, EventTrigger→TriggerAction, DataTrigger→Binding |
| **ImageSource методы** | FromFile→File…, FromUri→Uri…, FromResource→Resource…, FromStream→Stream… |
| **DatePicker vs TimePicker формат** | "D" (дата) vs "t" (время) |
| **Aspect: Fill** | единственное значение, которое НЕ сохраняет пропорции |
| **Border** | вмещает только ОДИН элемент |
